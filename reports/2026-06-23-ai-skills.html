⚙ 軟體設計哲學 Skills 精選
實質幫助工程師產出良好程式碼的設計哲學型技能 — 強調可讀性、可維護性、低耦合與高內聚，以書籍智慧為代理引擎注入工程靈魂。
語意感知哲學 — LSP × Agentic 設計 GitHub Copilot Claude Code
Semantic Awareness Type Safety Intent-Driven
今日 lsp-setup-skill 的落地代表一個新設計哲學的誕生：語意感知代理（Semantic-Aware Agent）。傳統 AI 代理靠文字相似度猜測程式碼意圖；語意感知代理透過 LSP 型別圖譜理解 函式實際做什麼、參數型別是什麼、哪裡被呼叫。

這一哲學的核心是：理解優先於模仿。代理不應透過「看起來像」來生成程式碼，而應透過「理解型別合約」來保證生成結果的正確性。lsp-setup-skill 讓 AI 工具鏈第一次具備真正的程式碼語意理解層，是 Agentic 工程從「語言模型生成」邁向「語意感知工程」的關鍵里程碑。

提供具體重構路徑將淺模組重塑為深模組（Deep Modules），讓介面簡單而實作封裝複雜性。MCPMarket 設計哲學類長期 #1。
