Review Loop 多輪品質哲學 Claude Code
Quality Gates Iterative Review Multi-Agent
今日升至 #13（進榜第 2 日即 +3 位）的 Review Loop，帶來的不只是一個工具，更是「代碼品質保障」的哲學重新定義：單次審查的局限性——即使是最好的 reviewer 也有認知盲點，每一輪審查角度都受前一輪發現影響，形成「確認偏誤鎖」。Review Loop 的多輪代理接力打破這個局限：每個子代理獨立啟動、不繼承前輪偏見，從不同視角切入（安全 / 效能 / 設計 / 可維護性），直到多輪迭代都無新發現，才判定審查完整。

這與審計原則深度對齊：重要系統不由單一審計者審查，而是多輪獨立稽核。SKILL.md Auto-Scan 進榜後可自動載入，零摩擦成為品質標準閘。
frontend-design — UI 意圖性哲學 Anthropic 官方 NEW
UI Intentionality Anti-Formulaic Design Communication
Anthropic 官方發佈的前端設計哲學 skill（280k+ 安裝），填補了 AI 工具鏈長期缺失的一塊拼圖：工具鏈有充分的能力幫助工程師寫出功能正確的 UI，卻幾乎沒有工具問「這個 UI 是否具有設計意圖」。

制式化陷阱：AI 生成的前端往往是 flexbox 置中 + 卡片堆疊 + 圓角陰影的機械組合，看似整齊，實則缺乏對「這個介面在向使用者傳達什麼」的思考。frontend-design 強制在每次 UI 生成前執行意圖審問：元素的視覺重量是否反映資訊優先級？布局節奏是否引導正確視線路徑？空白是呼吸還是浪費？

與 /review 配合時加入 UI 哲學審查維度，與 /simplify 配合時確保清理後仍保留設計意圖。設計不是裝飾，是溝通。

Karpathy Behavioral Claude Code
Goal Clarity Self-Doubt Loop Minimal Footprint
Andrej Karpathy 提出的 AI agent 行為哲學，直接對應工程師最常遭遇的代理失效模式。四大規則讓代理在執行前停下來問：「我真的理解這個目標嗎？」（Goal Clarity）、「我的每一步是否偏離了最初意圖？」（Self-Doubt Loop）、「我只動了必要的範圍嗎？」（Minimal Footprint）、「我能清楚說明我做了什麼嗎？」（Explicit Handoff）。

今日在 SKILL.md 設計場景下獲新詮釋：Minimal Footprint 延伸至 skill 設計哲學——Auto-Scan 自動發現的每個 SKILL.md 應精確定義單一職責，不攜帶超出任務範圍的副作用，讓「精確而非貪婪」在 skill 設計層面得以實踐。144k ⭐，Loop Engineering 生態核心行為護欄地位持續鞏固。
