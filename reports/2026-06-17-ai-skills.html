Loop Engineering Philosophy Claude Code Copilot NEW
Repeatable Loops STOP Conditions Memory Anchors
2026 年 6 月下旬確立的 AI 輔助開發設計哲學，核心論點：好的 AI 工程師不是「更快地提問」，而是「設計不需要一直提問的迴圈系統」。

三大設計原則：① 記憶錨點（Memory Anchors）——將任務上下文、邊界條件與 STOP 條件固化在 AGENTS.md，讓每次迴圈啟動都從可靠的基準出發；② 迴圈引擎（Loop Engine）——以 /loop 或排程機制驅動週期執行，而非手動觸發；③ 安全閘（Safety Gates）——明確定義迴圈終止條件，防止代理無限執行或偏離目標。

與軟體設計哲學的共鳴：Loop Engineering 是 Ousterhout「深模組（Deep Modules）」在系統層面的應用——將複雜的代理協作邏輯封裝在可重複迴圈架構內，對外只暴露簡單的啟動介面。今日空降排行 #5，討論增幅 520%。
Karpathy Behavioral Claude Code
Goal Clarity Self-Doubt Loop Minimal Footprint

refactor-clean (Spark) Claude Code
Refactor Code Smell DRY · KISS
專注識別並消除 Code Smell：深層巢狀、長參數列表、重複邏輯、命名語意模糊。強調 DRY（不重複自己） 與 KISS（保持簡單），讓後繼工程師能一眼理解業務意圖，而非破解實作細節。

與 /simplify skill 互補使用：/simplify 由 AI 自動修正，refactor-clean 提供背後的設計哲學方向盤，指導 AI「應該往哪個方向簡化」，在 Loop Engineering 迴圈收尾清理階段尤為關鍵。
