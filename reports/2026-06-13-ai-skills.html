⚙ 軟體設計哲學 Skills 精選
實質幫助工程師產出良好程式碼的設計哲學型技能 — 強調可讀性、可維護性、低耦合與高內聚，以書籍智慧為代理引擎注入工程靈魂。
software-design-philosophy Claude Code
Complexity Mgmt Deep Modules Cognitive Load
基於 John Ousterhout《A Philosophy of Software Design》，讓 Claude 在生成或審查程式碼時主動識別三大複雜度症狀：Change Amplification（修改一個地方需動十個地方）、Cognitive Load（讀者需要記憶太多上下文才能理解）、Unknown Unknowns（隱性假設埋設在程式碼深處）。

提供具體重構路徑將淺模組重塑為深模組（Deep Modules），讓介面簡單而實作封裝複雜性。MCPMarket 設計哲學類長期 #1。
agent-rules-books Claude Code Codex
Clean Code DDD Refactoring DDIA

核心理念：讓 AI 代理在生成任何程式碼前，先內化幾十年軟體工程智慧，而非只是完成眼前任務。支援所有 AGENTS.md 相容工具，可直接嵌入 CLAUDE.md / AGENTS.md。
clean-architecture-skills Claude Code
SOLID Clean Arch Dependency Rule
審查程式碼是否符合 Clean Architecture 的依賴規則與層次結構（Domain → Application → Infrastructure），識別跨層違規、God Class 與貧血模型。基於 Robert C. Martin 的架構理論，搭配 /review 的 --effort=high 效果最佳。MCPMarket 設計哲學類長期 #2。

SOLID Clean Code SRP · OCP · LSP

refactor-clean (Spark) Claude Code
Refactor Code Smell DRY · KISS

與 /simplify skill 互補使用：/simplify 由 AI 自動修正，refactor-clean 提供背後的設計哲學方向盤，指導 AI「應該往哪個方向簡化」。
